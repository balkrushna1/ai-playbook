I want you to build a production-ready MVP web application called "FlowPilot".

FlowPilot is a premium, modern, community-driven AI Workflow Directory platform.

IMPORTANT:
- This is NOT an automation tool.
- This is NOT a prompt marketplace.
- This is NOT a paid platform (all workflows are free).
- This is an open community where users can create and share structured AI workflows ("Playbooks").

Users should be able to:
- Sign up / log in
- Create playbooks
- View playbooks created by others
- Rate playbooks (1–5 stars)
- See average ratings publicly

The app must be clean, minimal, premium, and scalable.

====================================
CORE DATA STRUCTURE
====================================

Each workflow is called a "Playbook".

Playbook fields:

- id (uuid)
- user_id (foreign key to users)
- title (text)
- slug (unique text)
- short_description (text)
- category (text)
- difficulty (Beginner | Intermediate | Advanced)
- estimated_time (text)
- tools_used (array of text)
- created_at (timestamp)

Each Playbook contains multiple Steps.

Step fields:

- id (uuid)
- playbook_id (foreign key)
- step_number (integer)
- title (text)
- description (text)
- tool_used (text)
- prompt_text (text, optional)
- expected_output (text)

Rating fields:

Create a separate table for ratings:

Table: ratings
- id (uuid)
- playbook_id (foreign key)
- user_id (foreign key)
- rating (integer 1–5)
- created_at (timestamp)

Each user can rate a playbook only once.
Enforce unique constraint on (playbook_id, user_id).

Store average rating dynamically via query (do not store in table for MVP).
